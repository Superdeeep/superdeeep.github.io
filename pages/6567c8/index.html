Git的使用和踩坑

# Git的使用和踩坑

# 1、ssh

# 主要设置

先配置好全局用户

 git config --global user.name "你的名字"

 git config --global user.email "注册时使用的邮箱地址@qq.com"

接着生成密钥

 ssh-keygen -t rsa -C "注册时使用的邮箱地址@qq.com"

添加你的公钥


找到.ssh这个文件夹

Windows下通常在 /c/Users/你的用户名/.ssh/ 下

之后找到.pub的文件

在github上的deploy key里面添加你的公钥

链接到远程仓库

 git remote add origin1 https://github.com/XXXXXX/XXXXXXXXX.git

 git remote add origin2 https://github.com/XXXXXX/XXXXXX.github.io.git

::: info
ps:这里的 origin1 可以取你喜欢的名字(不过最好别这么搞)
:::


这步很重要不然会一直失败!!!
```git 
 echo "# 咔咔咔咔咔嘎嘎嘎嘎嘎" >> README.md

 git init

 git add README.md

 git commit -m "first commit"

 git branch -M main

::: info 如果是一步一步按照上面的方法做,却无法将已有的仓库推送到GitHub,还显示没有权限写入仓库的话,那么你或许跟我一样

把一个公钥用在了多台电脑上,或者没有配置好config

:::

# 附录

多仓库多用户

找到.ssh这个文件夹之后,找到或者新建config这个文件(它没有后缀名的)

Windows下通常在 /c/Users/你的用户名/.ssh/ 下

在config中进行如下配置:

host gitee.com

  Hostname gitee.com

  User 你的名字

  IdentityFile ~/.ssh/你的密钥名_rsa


host github.com

  Hostname github.com

  User 你的名字

  IdentityFile ~/.ssh/你的密钥名_rsa


host second.github.com

  Hostname github.com

  User 你的名字

  IdentityFile ~/.ssh/你的密钥名_rsa
  • ps: "host second.github.com" (这里的second可以取你喜欢的名字)
Git添加公钥命令(本地仓库) 
eval $(ssh-agent)

ssh-add ~/.ssh/publish_rsa
自动提交脚本 
git add -A
git commit -m 'deploy'
git push -f git@second.github.com:XXXXXX/XXXXX.github.io.git main
cd –

# 建立本地分支和仓库分支的概念

刚开始使用git的时候一定要重视本地分支和github上的分支，不然会出现推不上的情况，以下时相关命令

git push --set-upstream code master

git branch --set-upstream-to=code/master master

git branch -M master
